# 구현 로드맵

## Phase 1. 화면 구조 안정화

- 전체 피드 메신저형 UI 정리
- 하단 입력창 UX 확정
- 댓글/첨부 팝업 확정
- 참여자 패널 확정
- 모바일 메뉴 드로어 확정

## Phase 2. Supabase 연결

- Supabase 클라이언트 설정
- 게시판 목록 조회
- 피드 게시글 조회
- 게시글 작성
- 댓글 작성
- 첨부파일 Storage 연동

## Phase 3. 인증/권한

- Supabase Auth 연결
- 사용자 프로필 생성
- 팀별 게시판 접근 권한
- 대표/팀장 전체 열람 정책

## Phase 4. 업무게시판

- 기간 선택
- 목표/진행업무/이슈/다음 계획 입력
- 업무 기록 목록
- 전주 대비 Diff 표시 설계

## Phase 5. 개인 메모

- 개인 메모 작성
- 체크리스트
- 중요 표시
- 게시글 전환

## Phase 6. 알림

- 멘션 알림
- 댓글 알림
- 게시글 업데이트 알림
- 읽음/미읽음

## 현재 구현 상태

1. Supabase query 함수 기반으로 주요 화면이 동작한다.
2. 게시글 작성과 댓글 작성이 실제 `work_posts`, `work_comments` 테이블에 저장된다.
3. 전체 피드와 팀 게시판에서 첨부파일을 Supabase Storage `attachments` 버킷에 업로드한다.
4. 업무보고는 사용자/날짜 단위 upsert로 중복 제출을 방지한다.

## 남은 마무리 작업

1. 실제 Supabase 프로젝트에서 `supabase/schema.sql` 재실행 또는 마이그레이션 적용
2. 모바일 화면별 실제 사용성 점검
3. 알림 생성 트리거 또는 서버 액션 연동 고도화
