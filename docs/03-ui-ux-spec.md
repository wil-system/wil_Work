# UI/UX 기준

## 전체 피드

- 제목 입력 없이 본문 중심으로 작성한다.
- 카드형보다는 메신저형 말풍선 구조를 기본으로 한다.
- 최신 대화는 하단 입력창 가까이에 위치한다.
- 전체 피드에는 상태값을 표시하지 않는다.
- 게시글 메타 정보는 작성자, 소속, 시간, 핀 여부 정도만 표시한다.

## 하단 입력창

- ChatGPT 입력창처럼 하단에 떠 있는 형태로 표시한다.
- 별도 하단 패널처럼 보이지 않아야 한다.
- 기본 높이는 약 2줄이다.
- 입력 내용에 따라 높이가 자동 증가한다.
- 최대 높이 이후에는 입력창 내부만 스크롤된다.
- 첨부 아이콘은 입력창 내부 하단 왼쪽에 둔다.
- 등록 버튼은 입력창 내부 하단 오른쪽에 둔다.

## 댓글 팝업

- PC에서는 항상 화면 중앙에 표시한다.
- 팝업 왼쪽에는 원문 본문을 표시한다.
- 팝업 오른쪽에는 댓글 목록을 표시한다.
- 댓글 목록은 충분한 높이를 확보해 한 번에 여러 댓글이 보여야 한다.
- 댓글 입력창은 댓글 영역 하단에 유지한다.
- 댓글 팝업에는 파일첨부 기능을 넣지 않는다.
- 답글 버튼을 누르면 해당 댓글 아래에 답글 입력창이 표시된다.
- 팝업 바깥을 클릭하면 닫힌다.
- 팝업이 열려 있어도 본문 피드 스크롤은 막지 않는다.

## 첨부파일 팝업

- 댓글 팝업과 분리한다.
- 게시글 하단 첨부 버튼으로 연다.
- 파일명, 타입, 크기, 업로드 사용자를 표시한다.

## 참여자 패널

- 우측 토글 패널로 표시한다.
- 참여자는 소속별 그룹으로 표시한다.
- 부서명과 이름 목록은 시각적으로 명확히 구분한다.
- 검색, 전체/접속중 통계, 접속 상태값은 표시하지 않는다.

## 모바일

- 상단 게시판 가로 칩은 사용하지 않는다.
- `메뉴` 버튼 클릭 시 좌측 드로어가 열린다.
- 드로어는 PC 좌측 게시판 목록과 동일한 정보를 보여준다.
- 모바일에서 댓글/첨부는 화면 폭을 고려해 게시글 아래 인라인 표시를 기본으로 한다.
